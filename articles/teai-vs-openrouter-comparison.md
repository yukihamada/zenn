## はじめに

LLMアプリ開発で「モデルを自由に切り替えたい」「一つのAPIで複数プロバイダを使いたい」というニーズに応えるのがLLM API Gatewayだ。

このカテゴリで最も有名なのは **OpenRouter**（$5M ARR、250万ユーザー）。そして日本発の新しい選択肢として **teai.io** がある。

実際に両方を使って比較した。

## 基本スペック比較

| 項目 | OpenRouter | teai.io |
|------|-----------|---------|
| 本社 | 米国 | 日本（東京） |
| モデル数 | 290+ | 45+ |
| API形式 | OpenAI互換 | OpenAI互換 |
| マークアップ | 5.5% | **5%** |
| サーバー位置 | 米国 | **東京** |
| ドキュメント | 英語のみ | **日本語 + 英語** |
| 通貨 | USD | **JPY + USD** |
| インボイス制度 | 非対応 | **対応** |
| 無料モデル | あり（数モデル） | **Nemotron 9B無制限** |
| BYOK | なし | **あり** |

## レイテンシ比較

東京からの実測値（2026年3月、各10回の平均）。

### GPT-4o（非ストリーミング、短いプロンプト）

| メトリクス | OpenRouter | teai.io | 差 |
|-----------|-----------|---------|-----|
| DNS + 接続 | ~180ms | ~30ms | -150ms |
| TTFB | ~850ms | ~520ms | -330ms |
| 合計 | ~2,100ms | ~1,600ms | **-500ms** |

### Claude Sonnet 4.6（ストリーミング、最初のチャンク到着まで）

| メトリクス | OpenRouter | teai.io | 差 |
|-----------|-----------|---------|-----|
| First chunk | ~1,200ms | ~750ms | **-450ms** |

**なぜ差が出るか**: OpenRouterは米国サーバー経由。東京→米国→プロバイダ→米国→東京のラウンドトリップで150-300ms余分にかかる。teai.ioは東京→プロバイダ直接接続で1ホップ少ない。

## モデルカバレッジ

### OpenRouterの強み: 圧倒的なモデル数

290+モデル。ニッチなオープンソースモデルやコミュニティモデルまで幅広くカバー。

### teai.ioの強み: キュレーションと日本語最適化

45+モデルに絞り込み、すべて動作確認済み。特に：

## 開発者体験 (DX)

### コード移行

両方ともOpenAI SDK互換なので、移行は1行：

```python
# OpenRouter
client = OpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key="sk-or-..."
)

# teai.io
client = OpenAI(
    base_url="https://api.teai.io/v1",
    api_key="te_..."
)
```

### ドキュメント

- **OpenRouter**: 英語のみ。充実しているが、日本語話者には読むのにコストがかかる
- **teai.io**: 日本語ファースト。コード例も日本語コメント付き

## B2B/法人利用

**ここがteai.ioの最大の差別化ポイント。**

| 項目 | OpenRouter | teai.io |
|------|-----------|---------|
| 契約書 | 英語 | **日本語** |
| 請求書 | USD | **JPY（適格請求書）** |
| インボイス制度 | 非対応 | **対応** |
| サポート言語 | 英語 | **日本語** |
| SLA | なし | **99.9%（Pro以上）** |
| NDA | 英語 | **日本語** |

日本企業がOpenRouterを法人利用する場合、請求書の通貨変換、契約書の翻訳、経理処理の手間が発生する。teai.ioなら日本の商慣行に完全対応。

## こんな人にはOpenRouter

- 290+モデルの中からニッチなモデルを使いたい
- 英語ドキュメントで問題ない
- 個人利用でインボイス不要
- 暗号資産で支払いたい

## こんな人にはteai.io

- **日本在住**で低レイテンシが欲しい
- **法人利用**でインボイス制度対応が必要
- **日本語**のドキュメント・サポートが欲しい
- **無料**で始めたい（Nemotron 9B無制限）
- **BYOK**で自分のAPIキーを使いたい

## 結論

**OpenRouter** = グローバルスタンダード。モデル数と実績で圧倒的。英語圏にはベスト。

**teai.io** = 日本の開発者に最適化。レイテンシ、日本語対応、円建て請求、インボイス制度で差別化。特にB2B利用での優位性が明確。
